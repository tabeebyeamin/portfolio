Mock J Shell

In a group of 4, we created a file system and command prompt using Java for our Software Design Class. We planned our design using CRC Cards labelling each of the responsibilities and dependencies. Here are the principles we used: Scrum Meetings, Abstract Classes and Methods, Singleton Design Pattern, Single Responsibility Principle, Generics, Mock Objects

Scrum: held weekly scrum meetings to outline tasks for everyone to do

Abstract Classes/Methods: allowed us to better organize code

SRP: minimized responisbilities of each class

Mock Objects: these helped with unittesting. In order to test our classes, we didn't need to have the actual class done to test it.

What I learned

I learned that I enjoy working working with other programmers when we are communicating effectively. The agile scrum methodology we used helped me know exactly what to work on and what to expect from my team mates.
